Lesson 1: HTML Basics

HTML uses tags to define elements on a web page. Tags are enclosed in angle brackets and usually come in pairs: an opening tag and a closing tag.

Tag Syntax

Opening tag: <tagname>

Closing tag: </tagname>

Example: <p>This is a paragraph.</p>

Essential Text Tags

<h1> to <h6> - Headings

Define headings from most important (h1) to least important (h6).

<h1>Main Title</h1>

<p> - Paragraph

Defines a paragraph of text.

<p>This is a paragraph.</p>

<strong> and <em>

<strong>: Important text (bold)

<em>: Emphasized text (italic)

<strong>Bold</strong> and <em>italic</em>

<br> and <hr>

<br>: Line break (self-closing)

<hr>: Horizontal rule (self-closing)

Self-Closing Tags

Some tags don't need a closing tag because they don't contain content. These are called void elements.

Examples: <br>, <hr>, <img>, <input>

Lesson 2: Document Structure Tags

These tags define the structure and organization of your HTML document.

<html>

The root element that contains all other HTML elements.

<html lang="en">...</html>

<head>

Contains metadata about the document (not visible on page).

Includes: <title>, <meta>, <link>, <style>

<body>

Contains all visible content of the web page.

<body>...all visible content...</body>

List Tags

<ul> - Unordered List

Creates a bulleted list.

<ul><li>Item 1</li><li>Item 2</li></ul>

<ol> - Ordered List

Creates a numbered list.

<ol><li>First</li><li>Second</li></ol>

<li> - List Item

Defines an item in a list (used inside <ul> or <ol>).

Link and Media Tags

<a> - Anchor/Link

Creates hyperlinks to other pages or resources.

<a href="page.html">Click here</a>

<img> - Image

Embeds an image (self-closing).

<img src="photo.jpg" alt="Description">

Note: Always include alt attribute for accessibility.

Lesson 3: Semantic HTML Tags

Semantic tags clearly describe their meaning to both the browser and the developer. They improve accessibility and SEO.

Why Use Semantic HTML?

  • Better accessibility for screen readers
  • Improved SEO (search engines understand content better)
  • Easier code maintenance
  • Clearer document structure

Page Structure Tags

<header>

Introductory content or navigation links. Usually contains logo, site title, or navigation.

<nav>

Section containing navigation links.

<nav><a href="#home">Home</a></nav>

<main>

Main content of the document. Only one per page.

<article>

Self-contained content that could be distributed independently (blog post, news article).

<section>

Thematic grouping of content, typically with a heading.

<aside>

Content tangentially related to main content (sidebar, callout box).

<footer>

Footer of a document or section. Usually contains author info, copyright, links.

Content Tags

<figure> and <figcaption>

Used together to markup images, diagrams, or code snippets with captions.

<figure><img src="chart.png" alt="Sales chart"><figcaption>2024 Sales</figcaption></figure>

<code>

Displays a piece of computer code.

<code>let x = 5;</code>

Semantic vs Non-Semantic

Non-semantic: <div> and <span> - tell nothing about content

Semantic: <header>, <nav>, <article>, <footer> - clearly define content purpose
